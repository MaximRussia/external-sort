Простое решение для прохожения тестов из test.sh
Самописные тесты в test.sh
Оптимизация:
1) Использование более подходящих структур данных (list, heap и прочих) которые могут удалять из любого места за O(1), а не за O(n)
2) Многопоточность
3) Возможно есть какие-нибудь платформозависящие оптимизации
4) try-catch тормозит
5) Можно было бы использовать более сложный вариант, но более оптимизированный и овер-инженерный (требует исследований)
6) non-ascii на совести платформы
7) Возможно можно оптимизировать структуру chunk
8) Оптимизация работы с HDD. Баланс между размером чанки и их количеством.
